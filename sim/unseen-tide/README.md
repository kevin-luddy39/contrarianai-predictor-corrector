# Unseen Tide — predictor-corrector experiment

A 40-turn staged-perturbation test comparing four drift-detection strategies against a known-ground-truth contamination timeline. Companion to the whitepaper at [`../../docs/whitepaper-unseen-tide.md`](../../docs/whitepaper-unseen-tide.md).

## Running

```bash
cd tools/predictor-corrector
npm install
node sim/unseen-tide/runner.js
```

Outputs written to `results/`:

- `results.json` — full per-turn record (every statistic, every detector output)
- `summary.txt` — human-readable table of the session, plus a lead-time report

## What the experiment measures

Four detectors race on the same per-turn Context Inspector analysis:

| Detector | Alerts when |
|---|---|
| `static-mean` | observed domain mean falls below 0.50 |
| `static-sigma` | observed domain stdDev rises above 0.30 |
| `abm` | predictor-corrector health (ABM engine) falls below 0.72 |
| `kalman` | predictor-corrector health (Kalman engine) falls below 0.72 |

For each of four phases, we record the number of turns between the start of the phase and each detector's first alert.

## Expected output

Zero false positives in Phase 1 (calibration). Forecasters uniquely detect Phase 2 (stylistic drift) at T+7. Forecasters detect Phase 3 (adjacent creep) at T+1, static-σ at T+8. All detectors fire in Phase 4 (hard contamination).

See the whitepaper for full analysis.
